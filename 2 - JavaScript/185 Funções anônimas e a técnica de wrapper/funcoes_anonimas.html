Funções Anônimas

Uma função anônima em JavaScript é uma função que não possui um nome definido. Essas funções são frequentemente usadas quando você precisa de uma função temporária, especialmente como funções de callback ou dentro de expressões. A função anônima é geralmente atribuída a uma variável ou passada diretamente como argumento para outra função.

Exemplo de função anônima atribuída a uma variável:

javascript
Copiar código
var soma = function(a, b) {
return a + b;
};

console.log(soma(5, 3)); // Saída: 8
Exemplo de função anônima como callback:

javascript
Copiar código
setTimeout(function() {
console.log("Função anônima executada após 3 segundos!");
}, 3000);

Aqui, uma função anônima é passada como argumento para a função setTimeout, que a executa após 3 segundos.

Vantagens de Funções Anônimas

São úteis quando a função é usada apenas uma vez e não precisa ser referenciada posteriormente.
Simplificam o código em callbacks, como em eventos ou temporizadores.
Podem ser imediatamente invocadas, o que ajuda a encapsular código.
Wrapper Functions (Funções Wrapper)
Uma função wrapper ou função "invólucro" é uma função que "envolve" outra função, adicionando alguma lógica extra antes ou depois de executar a função original. Wrappers são úteis para modificar, melhorar ou adicionar funcionalidades sem alterar o código da função original.

Exemplo de função wrapper:

javascript
Copiar código
function logExecution(func) {
return function() {
console.log("Executando função...");
return func.apply(this, arguments);
};
}

var soma = function(a, b) {
return a + b;
};

var somaComLog = logExecution(soma);

console.log(somaComLog(2, 3)); // Saída: "Executando função..." e depois 5
Neste exemplo, logExecution é uma função wrapper que adiciona um console.log antes de executar a função passada como argumento. A função original (soma) não é alterada, mas sua execução agora é "envolvida" pela função wrapper.

IIFE (Immediately Invoked Function Expression)
Uma técnica muito usada com funções anônimas é o IIFE (Immediately Invoked Function Expression), que permite a criação de uma função anônima e sua execução imediata, sem precisar de uma referência posterior.

Exemplo de IIFE:

javascript
Copiar código
(function() {
console.log("Esta função foi executada imediatamente!");
})();

Essa técnica é usada para evitar que variáveis dentro da função poluam o escopo global, criando um escopo isolado.

Diferenças entre Função Anônima e Wrapper
Função Anônima: Uma função sem nome que pode ser usada como callback ou atribuída a uma variável.
Função Wrapper: Uma função que envolve outra função, adicionando comportamento extra antes ou depois de sua execução.

Essas duas técnicas podem ser usadas juntas ou separadamente, dependendo da necessidade de encapsulamento ou controle de escopos no código.
