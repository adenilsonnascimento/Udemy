Explicação sobre escopo de variáveis
Escopo define a área do código onde uma variável está disponível ou acessível. Em JavaScript, temos três tipos principais de escopo: escopo global, escopo de função e escopo de bloco.

1. Escopo Global
Variáveis declaradas fora de qualquer função ou bloco de código têm escopo global, o que significa que elas podem ser acessadas de qualquer parte do código, incluindo funções e blocos.

Exemplo no código:

javascript
Copiar código
var serie = 'Friends';
A variável serie é global e pode ser usada em qualquer parte do código.

2. Escopo de Função

Variáveis declaradas dentro de uma função têm escopo local à função. Isso significa que essas variáveis só podem ser acessadas dentro da função onde foram criadas. O escopo de função é ativado ao usar var, let ou const dentro de uma função.

Exemplo no código:

javascript
Copiar código
function x() {
var serie3 = 'The Walking Dead';
}

A variável serie3 só existe dentro da função x e não pode ser acessada fora dela. Tentar acessá-la fora da função resultará em um erro.

3. Escopo de Bloco

O escopo de bloco é ativado em blocos de código como if, for, ou {}. No entanto, var não respeita o escopo de bloco, ou seja, variáveis declaradas com var dentro de blocos são "levantadas" para o escopo global ou da função mais próxima. Já variáveis declaradas com let ou const respeitam o escopo de bloco.

Exemplo no código:

javascript
Copiar código
if (true) {
var serie2 = 'Game of Thrones';
}

Apesar de serie2 estar dentro do bloco if, ela pode ser acessada fora dele porque foi declarada com var. Se tivesse sido declarada com let ou const, ela só existiria dentro do bloco if.

Resumo sobre escopo:
Escopo global: Variáveis acessíveis de qualquer parte do código.
Escopo de função: Variáveis só podem ser acessadas dentro da função onde foram declaradas.
Escopo de bloco: Com let e const, as variáveis só existem dentro do bloco onde foram declaradas. Com var, elas "escapam" para o escopo global ou de função.
Esses conceitos são fundamentais para evitar erros, como tentar acessar variáveis que estão fora do seu escopo, ou sobrescrever variáveis acidentalmente.
